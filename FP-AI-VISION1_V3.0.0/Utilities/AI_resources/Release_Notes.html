Purpose

AI_Resources contains resources needed by the user to:

  • Quantize float model and regenerate C model from X-CUBE-AI V5.1.2 tool
  • Re-train the provided neural network models
  • Convert image files to the BMP format expected by the “on-board validation” library

Here is the list of references to user documents:

  • UM2526 : Getting started with X-CUBE-AI Expansion Package for Artificial Intelligence (AI)

Update History

Main Changes

  • Adding scripts enabling to retrain the Person Presence Detection MobileNetV2 neural network model.
  • Updated script for image file format conversion.

Contents

  • Files (test_set_generation_foodreco_nn.py and config_file_foodreco_nn.json) required by the quantization process as described in UM2526.
  • Food Recognition floating point model (FoodReco_MobileNet_Derivative_Float.h5 file). This file can be used in order to regenerate the float C model with X-CUBE-AI V5.1.2.
  • Food Recognition quantized model (FoodReco_MobileNet_Derivative_Quantized.h5/.json) generated using X-CUBE-AI and using “UaUa per layer” quantization scheme (for details refer to UM2526). This file can be used in order to regenerate the quantized C model with X-CUBE-AI V5.1.2.
  • Person Presence Detection quantized models (person_detect.tflite and mobv2_128x128x3_adam02_respin01.best_val.035.h5.uint8.tflite files). Those files can be used in order to regenerate the quantized C models with X-CUBE-AI V5.1.2.
  • FoodDetection.py/.ipynb files enabling the retraining of the Food Recognition neural network model.
  • create_dataset.py file for converting image file format to BMP format (format expected by the “onboard validation” library).
  • Scripts for retraining Person Presence Detection MobileNetV2 neural network model.

Main Changes

  • Updated quantized model generated using X-CUBE-AI V5.0.0.
  • New script enabling to retrain the provided neural network model.
  • New script for image file format conversion.

Contents

  • Files (test_set_generation_foodreco_nn.py and config_file_foodreco_nn.json) required by the quantization process as described in UM2526.
  • FoodReco_MobileNet_Derivative_Float.h5 file that correspond to Keras saved model of the Food recognition neural network. Those files can be used in order to regenerate the float and quantized C model with X-CUBE-AI V5.0.0.
  • Quantized model (FoodReco_MobileNet_Derivative_Quantized.h5/.json) generated using X-CUBE-AI V5.0.0. and using “UaUa per layer” quantization scheme (for details refer to UM2526).
  • FoodDetection.py/.ipynb files enabling the retraining of the neural network model.
  • create_dataset.py file for converting image file format to PPM P6 format (format expected by the “onboard validation” library).
